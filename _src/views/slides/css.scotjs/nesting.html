Looking at preprocessors it's very tempting to think that we can use them to create name spaces by nesting code, after all this looks a lot like block scope in curly brace languages. However, just because it looks like something you are familiar with it doesn't mean that it actually works that way. The important thing with preprocessors is to think about them in terms of the output, not the input. You're still writing CSS that will be delivered to the browser. There's no compiler doing any optimisations, you're just writing CSS indirectly.
