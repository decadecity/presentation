!important

.error {
  color: red;
}
.email .error {
  color: blue;!important;
}
#login-form .error {
  color: orange;
}
We can't up the specificity of the email field selector so we'll add an `!important` annotation: >>>> The way `!important` works is by resetting the cascade. Effectively there's now a second cascade of declarations that have the `!important` annotation so here the `.email .error` colour will now trump the higher specificity selector for the login form because that doesn't have an `!important` annotation.
